Rentcar App — каталог оренди авто (Next.js)

Огляд

- Next.js + TypeScript застосунок для перегляду і оренди авто.
- Підтримує фільтри (бренд, ціна, пробіг), перегляд деталей авто, обране, пагінацію та форму бронювання.
- Дані отримуються з публічного API: https://car-rental-api.goit.global/api-docs/

Можливості

- Каталог: перегляд списку авто з кнопкою Load More та пагінацією на бекенді.
- Фільтри: бренд (All brands), ціна за годину, пробіг (from/to з підтримкою ком/крапки).
- Пошук: одна кнопка Search застосовує всі активні фільтри одразу.
- Обране: додавання авто у вибране (зберігається у Zustand).
- Деталі: сторінка авто з фото, повною інформацією та формою бронювання.

Маршрути ⛗

- `/` — домашня сторінка з Hero та переходом до каталогу.
- `/catalog` — перелік авто з фільтрами та пагінацією.
- `/catalog/[id]` — сторінка конкретного авто з деталями та формою.

Технології ⚒️

- Next.js App Router, React, TypeScript
- Axios (HTTP), Zustand (стан)
- CSS Modules для стилів

Встановлення та запуск

```zsh
git clone <repo_url>
cd "Rentcar application/rentcar-app"
npm install
npm run dev
```

Виробнича збірка:

```zsh
npm run build
npm start
```

Налаштування фільтрів (коротко)

- Компонент: `app/components/FilterBar/FilterBar.tsx`
- Кнопка `Search` оновлює URL-параметри: `brand`, `price`, `mileageFrom`, `mileageTo`.
- Опція `All brands` очищає фільтр бренду (значення `""`).
- Поля пробігу підтримують формат з комою, значення нормалізуються до числа.

API

- Сервіс: `app/services/api.ts`
- `getCars(page, limit, filters)` — отримання списку авто.
- `getCarById(id)` — дані конкретного авто.
- `getBrands()` — перелік брендів.

Стан та обране

- Стор: `app/store/useCarsStore.ts`
- Зберігає масив `favorites` та методи додавання/видалення.

Структура каталог/деталі ⚙️

- Каталог: `app/catalog/page.tsx` (сервер) + `app/catalog/CatalogClient.tsx` (клієнт).
- Деталі: `app/catalog/[id]/page.tsx` + `app/components/CarDetails/CarDetails.tsx`.

Стандарти коду
